Hyperion 6-DOF Engine

Flight Software Infrastructure

System Overview

Hyperion is a ground-up C++20 physics engine designed for Hardware-in-the-Loop (HIL) validation of flight software. Unlike game engines, it prioritizes deterministic timing and thread safety to simulate orbital mechanics and rigid body dynamics.

  • Multithreaded Architecture: Decouples high-frequency physics (10kHz) from low-frequency telemetry (50Hz) using std::thread and std::mutex locking strategies.
  • Custom Integrator: Implements a 4th-Order Runge-Kutta (RK4) solver from scratch to handle orbital precision without the overhead of external math libraries.
  • UDP Telemetry: Serializes the state vector into binary packets and broadcasts via BSD sockets, mimicking real spacecraft-to-ground communication.

Vec3 compute_acceleration(const State& state) {
    // 1. Basic Monopole Gravity: F = -mu/r^3 * r_vec
    double r_sq = state.pos.dot(state.pos);
    double mu_r3 = MU_EARTH / (r_sq * std::sqrt(r_sq));
    
    // 2. J2 Perturbation (Earth Oblateness)
    double z2 = state.pos.z * state.pos.z;
    double j2_factor = 1.5 * J2 * MU_EARTH * (R_EARTH * R_EARTH) / (r_sq * r_sq);
    
    Vec3 g_j2 = {
        j2_factor * (state.pos.x / r_mag * (5 * z2 / r_sq - 1)),
        j2_factor * (state.pos.y / r_mag * (5 * z2 / r_sq - 1)),
        j2_factor * (state.pos.z / r_mag * (5 * z2 / r_sq - 3))
    };

    return g_monopole + g_j2;
}
src/main.cpp Thread Synchronization 
void physics_loop() {
    double dt = 0.0001; // 100us Time Step
    
    while (running) {
        // 1. Compute Dynamics (Heavy Math)
        Vec3 forces = compute_forces(local_state);
        rk4_integrate(local_state, forces, dt);

        // 2. CRITICAL SECTION: Update Shared State
        {
            std::lock_guard<std::mutex> lock(state_mutex);
            shared_state = local_state;
        }

        // 3. Precision Timing (Spinlock)
        spin_wait_until(next_tick);
    }
}
