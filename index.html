Incident #VTOL-01

Sensor Dropouts on Bench Test

The Issue

GPS and magnetometer were intermittently disconnecting during bench testing.

Root Cause

Electrical noise from the high-current motor wiring interfered with the sensor cables.

The Fix

Re-routed and shielded the avionics wiring, eliminating sensor dropouts.

Resolved
Incident #HYPERION-01

Thread Contention Causing Physics Jitter

The Issue

The 10 kHz physics loop occasionally produced small timing spikes, creating slight jitter in the simulated orbit.

Root Cause

The physics and telemetry threads were competing for the shared state mutex, delaying some RK4 integration steps.

The Fix

Introduced double-buffered state handoff, eliminating mutex blocking and restoring stable 10 kHz timing.

Resolved
Incident #DBF-01

Borderline Pitch Stability

The Issue

The aircraft tended to “hunt” in pitch during cruise instead of settling into a steady attitude.

Root Cause

The final battery and payload placement moved the center of gravity farther aft than the original CAD-based estimate.

The Fix

Shifted mass forward and increased tail authority until pitch stability margins were comfortably within the design target.

Resolved
Incident #WEBTUNNEL-01

Simulation Performance Slowdown

The Issue

A browser-based simulation dropped from 60 FPS to about 5 FPS after ~20 seconds of runtime.

Root Cause

The code was allocating new arrays every frame instead of reusing existing memory, slowly filling up the heap.

The Fix

Switched to reusing shared buffers each frame, which restored stable frame rates.

Resolved
Incident #FLIGHTNAV-01

EKF Timing Drift in Embedded Loop

The Issue

The EKF loop occasionally ran slower than intended, causing small timing offsets between IMU, GPS, and barometer updates.

Root Cause

Extra matrix allocations in the Eigen code path created micro-delays, especially when GPS updates arrived during the predict cycle.

The Fix

Preallocated matrices and separated the GPS/Baro update path, restoring consistent loop timing on the Teensy.

Resolved
Incident #MIROLAB-01

EMG–MMG Onset Offset

The Issue

EMG and MMG signals for the same muscle contraction appeared about 100 ms apart.

Root Cause

The two channels were recorded on different hardware paths that introduced different delays into each signal.

The Fix

Added a shared trigger and timestamp alignment so both channels are synchronized in the final dataset.
